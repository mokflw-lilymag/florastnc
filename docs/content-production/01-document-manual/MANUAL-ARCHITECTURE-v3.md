## 1. 사장님 아이디어에 대한 의견

### 1.1 결론 — **매우 좋은 방향입니다. 적극 추천합니다.**

꽃집 사장님·알바는 보통 이렇게 묻습니다.

- “리본 프린터 어디서 켜?” (메뉴 질문)
- “휴대폰으로 주문 넣었는데 왜 안 나와?” (상황 질문)
- “배송 끝났는데 배송비 지출은 어디에 찍혀?” (업무 흐름 질문)

**메뉴만 나열한 백과사전**은 나중에 찾기엔 좋지만, **처음·급할 때**는 답이 늦습니다.  
**상황별 가이드**는 “지금 당장 할 일”에 맞춰져 있어 교육·CS·셀프서비스 모두에 유리합니다.

그래서 **한 HTML 안에 둘 다 두되, 역할을 분리**하는 것이 최선입니다.

| 축 | 비유 | 언제 쓰나 |
|----|------|-----------|
| **전체 메뉴 백과** (Track M) | 책의 **색인·항목별 설명** | “이 화면이 뭐하는 거지?” |
| **상황별 가이드** (Track S) | **레시피·체크리스트** | “지금 이거 하고 싶어!” |

### 1.2 주의할 점 (우리가 설계로 막을 것)

| 리스크 | 대응 |
|--------|------|
| 같은 내용이 두 번 길어짐 | 상황 가이드 = **짧은 순서**만, 자세한 필드 설명은 Track M으로 **링크** |
| 목차가 복잡해짐 | 맨 위 **허브 한 페이지** + 사이드바 **상단에 상황별** 고정 |
| 앱 업데이트 시 어긋남 | 상황 가이드마다 `관련 메뉴:` · `최종 검수일` 표기 |
| 모바일/PC 차이 혼동 | 각 가이드 상단에 **🖥️ PC만** / **📱 모바일 가능** 배지 |

### 1.3 v2에서 이미 한 일과의 관계

v2(사용자 친화 언어, 윈도우/안드로이드, `#settings-printer`, `#appendix-apps`)는 **Track M·부록**에 그대로 둡니다.  
v3는 그 위에 **Track S 전체 층**을 얹는 작업입니다.

## 4. 상황별 가이드 목록 (1차 — 사장님 제안 + 확장)

### Tier S1 — 필수 (1~2주, 사장님이 직접 언급)

| ID | 앵커 | 제목 (사이드바 표기) | 핵심 상황 | PC/모바일 |
|----|------|----------------------|-----------|-----------|
| S01 | `#sit-pos-printer` | **영수증·주문서 인쇄하기** (휴대폰 주문도 매장에서 출력) | 웹만 쓰는 매장 인쇄 연결 · 외부 접수 자동 인쇄 | 🖥️ 설치·설정 PC / 📱 주문 접수만 |
| S02 | `#sit-ribbon-printer` | **리본 프린터 설정하기** | 환경 설정 스위치 + 리본 프린터 메뉴 · **모바일에서는 리본 인쇄 불가** | 🖥️ **PC만** |
| S03 | `#sit-delivery-fee` | **배송비 규칙 설정하기** | 지역별 배송비 · 기본 배송비 · 무료 배송 기준 | 🖥️ 설정 PC / 📱 주문 시 자동 반영 |
| S04 | `#sit-quick-pos` | **빠른 POS — 매장에서 바로 결제** | 현장 고객 · 카드/현금 등 선택 → **결제완료·주문완료** 한 번에 | 🖥️·태블릿 |
| S05 | `#sit-pickup-delivery` | **픽업·배송 마무리** (실제 배송비·결제·지출까지) | 배송 화면에서 실배송비 입력 → 배송완료 · 고객 결제수단 반영 · **지출에 배송비 기록** | 🖥️ 권장 |

#### S01 상세 개요 (작성 시 포함할 흐름)

```
[휴대폰·다른 PC] 주문 접수
        ↓
   (클라우드에 저장)
        ↓
[매장 PC] 환경 설정 → 프린터/브릿지 「연결됨」
        ↓
   영수증·주문서 자동 출력
```

- 윈도우 앱 쓰는 매장: 별도 연결 프로그램 생략 → `#appendix-apps` 링크  
- **쓰지 않을 말:** 포트, 폴링, print_jobs  

#### S02 상세 — 모바일 불가 (명확히 박스로)

> **📱 안드로이드·모바일 웹에서는 리본 인쇄를 할 수 없습니다.**  
> 리본은 **매장 PC**에서 **리본 프린터** 메뉴로만 출력하세요.

#### S03 상세 — 배송비

**어디로:** 환경 설정 → **배송** 탭  

| 화면 항목 | 설명 |
|-----------|------|
| 기본 배송비 | 주소 매칭 안 될 때 쓰는 기본값 |
| 무료 배송 기준 | 이 금액 이상이면 배송비 0 |
| 지역별 배송비 | 구/동 이름 + 금액 추가 |

**주문할 때:** 새 주문 · 모바일 주문에서 주소 넣으면 **자동** 배송비 (수동 전환 스위치 있음)

#### S04 상세 — 빠른 POS

**어디로:** (앱 내 진입 경로 코드 확인 후 확정 — 예: 모바일 주문 화면 링크 또는 직접 `/dashboard/pos/quick`)

**흐름:** 상품 담기 → 할인(선택) → **결제 수단** (카드·현금·계좌이체 등) → **[결제 완료]**  
**결과:** 주문 상태 **완료**, 결제 **결제완료**, 매장 수령(store) 유형 — 배송·실배송비 단계 **없음**

#### S05 상세 — 픽업·배송 “한 판 끝내기” (사장님 요청 핵심)

**이 가이드가 답하는 질문:**  
“배송 다 했는데, 결제·주문완료·실제 택배비 지출까지 어떻게 한 번에 정리해?”

**배송 예약 주문 — 권장 마무리 순서**

| 단계 | 어디서 | 하는 일 | 시스템에 남는 것 |
|------|--------|---------|------------------|
| 1 | **새 주문** | 고객·상품·배송지·**배송비(자동/수동)**·리본 | 주문 생성 |
| 2 | 저장 시 | 결제 수단·**결제완료/미수** 선택 | `payment.status` |
| 3 | **배송 · 픽업** (`/dashboard/delivery`) | 해당 주문 찾기 | 일정 확인 |
| 4 | 같은 화면 | **실제 배송비** (카드·현금 분리 입력 가능) 저장 | `actual_delivery_cost` · **지출(배송비) 자동 반영** |
| 5 | | 배송 상태 **완료** 처리 | 배송완료 |
| 6 | (선택) | **일일 마감 정산** | 당일 결제·미수·배송비 지출 집계 |

**픽업 예약:** 4단계 실배송비 생략 가능, 2·5·6은 동일하게 적용.

**한 줄 요약 박스 (매뉴얼에 넣을 문구):**  
> 결제 수단을 고르고 저장하는 순간 **주문·결제 상태**가 정해지고, 배송 화면에서 **실제 택배비**를 넣으면 **지출 장부**까지 이어집니다.

**연결 Track M:** `#nav-new-order`, `#nav-delivery`, `#nav-daily-settlement`, `#nav-expenses`

## 4.1 발주·이관 용어 (정본)

| 기능 | 대상 | DB·화면 | 매뉴얼 앵커 |
|------|------|---------|-------------|
| **외부발주** | FloXync 미가입 거래처 | `outsource_info` · 주문 ⋮ → 외부 발주 | `#sit-outsource` · `#nav-outsource` |
| **지점 이관** | 같은 `organization` 지점 | `order_transfers` · 지점 이관 요청 | `#hq-branch-transfer` (요약 `#nav-branch-transfer`) |
| **회원사 수발주** | FloXync 가입 다른 tenant | `external_orders` · 회원사 수발주 발주 | `#nav-partner-orders` |

**폐기 용어:** 「협력사 수발주」(도매·광고주 링크 구상) — 기능 없음. UI·설정에서는 **회원사 수발주**로 통일. `platform_config.partner_orders_enabled`는 회원사 수발주 on/off.

비교표: `#nav-order-flows` · 용어집: `_shared/glossary.md`

상황 가이드가 **덮지 않는** 영역은 Track M이 담당합니다. 기존 HTML 절을 유지·보강합니다.

| 장 | 앵커 | 상황 가이드와 관계 |
|----|------|-------------------|
| 허브 | `#manual-hub` | 신규 |
| 환경 설정 | `#settings-*` | S01·S02·S03이 요약, M이 상세 |
| 주문 | `#orders-*` | S04·S05·S08 연결 |
| CRM·매출 | `#crm`, `#revenue-engine` | S09·S10 |
| 일상 메뉴 | `#nav-*` | 메뉴 하나 = 한 article, 상단에 `빠른 가이드: #sit-…` |
| **다지점·본사** | `#hq-org-start`, `#hq-org-roles`, `#hq-org-menu`, `#hq-org-team`, `#hq-branch-transfer` … | 조직 온보딩(운영팀 연결) · 역할·메뉴 · 지점 이관은 지점 업무 하위 |

**각 `#nav-*` article 상단에 넣을 한 줄 (템플릿):**
