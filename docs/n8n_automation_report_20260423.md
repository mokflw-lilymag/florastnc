# 📊 N8N 마케팅 자동화 연동 작업 보고서 (2026-04-23)

## 📋 1. 작업 개요
본사 SaaS(Floxync) 플랫폼에서 7개 소셜 미디어 채널(Instagram, Facebook, Threads, TikTok, YouTube, Blogger, Naver)로 콘텐츠를 자동 발행하기 위한 n8n 마스터 워크플로우 디버깅 및 최종 최적화 작업을 수행함.

## 🛠 2. 주요 개선 및 해결 사항

### ✅ 워크플로우 표준화 (v19/v20)
- **HTTP Request 노드 통합**: 기존 UI 기반 설정의 불안정성을 제거하기 위해 모든 노드를 `n8n-nodes-base.httpRequest` 버전 4로 교체 및 표준화.
- **데이터 구조 수정**: n8n v4의 새로운 규격에 맞춰 `bodyParameters.parameters`와 `headerParameters.parameters`를 명시적으로 설정하여 데이터 누락 방지.
- **이미지 주소 매핑**: SaaS 백엔드에서 전송하는 `image_url` 필드와 메타 API의 요구 필드를 완벽하게 일치시킴.
- **7개 채널 완전 연결**: 모든 플랫폼 노드에 고유 ID를 부여하여 불러오기 시 연결선이 끊어지는 문제 해결.

### ✅ 보안 및 인증 최적화
- **Supabase 연동**: 전역 API 키와 개별 유저 토큰을 DB에서 동적으로 불러오는 로직 강화.
- **토큰 유형 정정**: 일반 '사용자 액세스 토큰'의 한계를 파악하고, **'페이지 액세스 토큰(Page Access Token)'**으로 교체하여 관리 권한 문제 해결.
- **실시간 DB 업데이트**: 유저가 발급받은 최신 메타 토큰을 `user_credentials` 테이블에 즉시 반영.

## ⚠️ 3. 현재 남은 과제 (Blocker)

### 🚨 인스타그램 비즈니스 계정 연결 필요
- **현상**: 최신 페이지 토큰을 사용해도 인스타그램 발행 시 `instagram_business_account` 필드를 찾을 수 없다는 에러 발생.
- **원인**: 현재 'Flora Sync' 페이스북 페이지와 인스타그램 계정 사이의 물리적인 **비즈니스 연결**이 되어 있지 않음.
- **해결 방법**: 
  1. 페이스북 페이지 설정 -> [연결된 계정] -> [Instagram]에서 계정 연결 수행.
  2. 인스타그램 계정이 반드시 **'프로페셔널(비즈니스)'** 유형이어야 함.

## 🚀 4. 향후 실행 가이드
1. **페이스북 테스트**: 현재 설정으로 `Pub_Facebook` 노드는 즉시 발행 가능함. 먼저 성공 여부 확인 권장.
2. **인스타그램 연결**: 위 3번 항목의 연결 설정 완료 후 n8n 워크플로우 재실행.
3. **토큰 갱신**: 현재 토큰은 단기용이므로, 장기적으로 60일짜리 장기 토큰(Long-lived Token)으로의 교체 필요.
