# 카페24 × 플록싱크 — 주문 자동 수집 완전 가이드

> **한 줄 요약:** 카페24 개발자센터에서 앱과 `mall.read_order`(주문 읽기) 권한을 만든 뒤, 플록싱크 **환경 설정 → 연동 및 자동화**에 키를 넣고 **카페24 로그인 연동**으로 OAuth만 마치면, **[주문 관리]**에서 **쇼핑몰 동기화**로 주문을 끌어올 수 있습니다.

| 구분 | 마스터 매뉴얼 (`docs/floxync-manual.html`) | 이 문서 |
|------|---------------------------------------------|----------|
| 역할 | 화면 위치·흐름을 **한눈에** (1-6 연동 및 자동화) | 카페24만 **끝까지** 따라 하기 |
| 깊이 | 4줄 요약 + 주의 각주 | 단계별 체크리스트·막힘 해소·보안 습관 |

**공개 매뉴얼에서 같은 섹션 보기:** 배포 주소 기준 **`/docs/manual#settings-automation`** → 카드 **Cafe24 연동** (`#integration-cafe24`).  
예: `https://floxync.com/docs/manual#integration-cafe24` · 로컬 개발 시 `http://localhost:3000/docs/manual#integration-cafe24`

---

## 시작하기 전에 (1분 체크)

- [ ] 카페24 **자사몰** 관리자 계정(쇼핑몰 ID)을 알고 있습니다.  
  → 플록싱크에 넣는 **Mall ID**는 보통 `xxx.cafe24.com`의 **`xxx`** 부분입니다. (예: `lilymagflower.cafe24.com` → Mall ID `lilymagflower`)
- [ ] 브라우저 **팝업 차단**을 잠시 해제할 수 있습니다. (OAuth 창이 뜹니다)
- [ ] **Client ID / Secret**은 카톡·캡처로 보내지 말고, 메모장에만 잠깐 두었다가 입력 후 지웁니다.

---

## 1단계 — 카페24 개발자센터에서 앱과 “열쇠” 만들기

카페24가 플록싱크에 **“이 몰의 주문을 읽어도 된다”**고 허락하려면, 개발자센터에서 앱을 등록하고 **Client ID + Client Secret**을 발급받아야 합니다.  
(매뉴얼에 적힌 **관리자 → 앱스토어** 경로는 몰 버전에 따라 다를 수 있어, 아래는 **developer.cafe24.com** 기준으로 통일합니다.)

1. **[카페24 개발자센터](https://developer.cafe24.com/)** 접속 → 쇼핑몰 관리자 계정으로 로그인  
2. 상단 **나의 앱** → **앱 만들기**  
3. 앱 이름은 본인만 보이므로 `플록싱크 주문연동` 등으로 자유롭게  
4. **API 권한** 탭에서 반드시 **`mall.read_order` (주문 읽기)** 를 켭니다.  
   - 주문만 가져오면 되는 기본 연동에는 이 권한이 핵심입니다.  
   - (상품·재고까지 나중에 확장할 계획이면 카페24 정책에 맞게 추가 권한을 검토하세요.)  
5. 앱 정보 화면에서 **Client ID**, **Client Secret** 을 복사해 안전한 곳에 잠시 보관합니다.

---

## 2단계 — 플록싱크에 Mall ID와 키 넣기

앱 화면의 메뉴 이름은 다음과 같습니다. (가이드 초판의 **「쇼핑몰 연동 관리」** 표현은 여기로 통일합니다.)

1. 플록싱크 로그인 → 좌측 **환경 설정** (톱니바퀴)  
2. **연동 및 자동화** 탭으로 이동  
3. 카드 **「쇼핑몰 주문 자동 수집 (API 연동)」** 안에서 **카페24** 영역을 찾습니다.  
4. **카페24 개발자 API** 스위치를 **켜기**합니다.  
5. 입력란을 채웁니다.  
   | 항목 | 설명 |
   |------|------|
   | **쇼핑몰 아이디 (Mall ID)** | `쇼핑몰주소.cafe24.com` 앞부분 (하이픈 없이) |
   | **Client ID** | 1단계에서 복사한 값 |
   | **Client Secret** | 1단계에서 복사한 값 |
6. **인증 테스트 및 저장** 을 누릅니다.  
   - 저장이 되면 토스트로 안내가 뜨고, **실제 수집을 하려면 다음 단계 OAuth(로그인 연동)** 가 필요하다는 메시지가 이어질 수 있습니다.

---

## 3단계 — 카페24 로그인 연동 (OAuth)

키만으로는 주문 API를 호출할 수 없습니다. 쇼핑몰 **관리자가 한 번 동의**해 주어야 액세스 토큰이 발급됩니다.

1. 같은 화면에서 **카페24 로그인 연동** 버튼을 누릅니다. (이미 연동된 적이 있으면 **재인증**으로 표시됩니다.)  
2. **Mall ID가 비어 있으면** 버튼을 눌렀을 때 오류가 납니다. 2단계에서 반드시 입력했는지 확인하세요.  
3. 새 창(또는 팝업)에서 카페24 로그인·**권한 동의**를 완료합니다.  
4. 리다이렉트는 플록싱크 서버의 콜백으로 이어집니다. (구현상 `https://floxync.com/api/sync/cafe24/callback` 등 운영 도메인을 사용합니다.)  
5. 성공 후 창을 닫고, 플록싱크를 **새로고침**하면 **연동됨** 배지·**주문 수동 동기화** 버튼이 보이기 시작할 수 있습니다.

---

## 4단계 — 주문 가져오기

1. 좌측 메뉴 **주문 관리** (또는 **주문 목록**, `/dashboard/orders`)로 이동합니다.  
2. 상단 **쇼핑몰 동기화** (웹에서만 보일 수 있음)를 눌러 카페24 주문을 끌어옵니다.  
3. 설정 화면 카페24 블록에 **주문 수동 동기화**가 보이면, 거기서도 동기화를 시도할 수 있습니다.

---

## “14일의 마법” — 왜 가끔만 다시 로그인하면 되나요?

카페24는 보안을 위해 **액세스 토큰의 수명을 짧게** 주고, 대신 **리프레시 토큰(긴 유효기간)** 으로 새 액세스 토큰을 받도록 설계하는 경우가 많습니다. 플록싱크는 그 흐름을 살려 **자동 갱신**을 시도합니다.

- **짧은 수명의 액세스 토큰**이 끝나면, 서버가 리프레시 토큰으로 **새 액세스 토큰**을 요청합니다.  
- **매장에서 플록싱크를 주기적으로 사용**하면 갱신이 이어져, 매번 쇼핑몰 관리자 로그인을 반복하지 않아도 되는 것이 목표입니다.  
- **오랫동안(예: 약 2주 가까이) 앱에 한 번도 접속하지 않으면** 리프레시 토큰도 만료될 수 있습니다. 그때는 **3단계와 같이 재인증** 한 번으로 다시 연장하면 됩니다.

> 정확한 분·시간은 카페24 정책과 앱 버전에 따라 바뀔 수 있으니, 동기화가 갑자기 안 되면 **재인증 → 저장 → 쇼핑몰 동기화** 순서를 먼저 시도해 보세요.

## 막힘 해소 (FAQ)

| 증상 | 점검할 것 |
|------|-----------|
| 동기화 버튼이 안 보인다 | **모바일/앱 WebView**에서는 숨겨질 수 있습니다. **PC 브라우저**에서 주문 목록을 열어 보세요. |
| OAuth 창이 안 뜬다 | **팝업 차단** 해제, Mall ID·Client ID 입력 여부 |
| “연동됨”인데 주문이 0건 | 카페24에 **새 결제 주문**이 있는지, 날짜·몰이 맞는지 확인 |
| 토큰 오류 메시지 | **재인증** 후 다시 **인증 테스트 및 저장** |

## 보안 습관 (꼭 읽어 주세요)

- Client Secret은 **비밀번호와 동급**입니다. 카톡·이메일 본문에 넣지 마세요.  
- 스크린샷으로 올리기 전에 **키 부분을 가리기** 마스킹 하세요.  
- 유출이 의심되면 카페24 개발자센터에서 **시크릿 재발급** 후 플록싱크에 다시 입력하세요.
