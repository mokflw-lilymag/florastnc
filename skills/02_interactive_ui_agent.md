# 안티그래비티 스킬: [인터랙티브 UI] "누끼 따고 드래그하고"

## 사용 목적
`@dnd-kit`이나 `@imgly` 처럼 브라우저 자원을 많이 쓰는 인터랙티브 UI 기능을 구현할 때 필수적으로 사용합니다.

## 시스템 프롬프트
"사용자가 이미지를 업로드하면 **@imgly/background-removal**로 배경을 지우고, 그 결과물을 **@dnd-kit**이 적용된 캔버스 위에 배치해야 해. 이 과정에서 발생하는 레이턴시를 최소화하기 위해 **IndexedDB(idb-keyval)**를 어떻게 캐시로 활용할 수 있을까? 성능 최적화(Memoization)를 고려한 React 컴포넌트 구조와 에러 핸들링 로직을 코드로 짜줘."

## 핵심 가이드라인
* 브라우저 메인 스레드 부하를 줄이는 렌더링 최적화를 제안할 것 (Off-the-main-thread).
* `idb-keyval` 등 브라우저 내장 스토리지를 활용한 영리한 캐싱 전략 포함.
* 오류 상황(AI 로딩 실패 등)에 대한 Fallback UI 및 에러 핸들링 구성.
