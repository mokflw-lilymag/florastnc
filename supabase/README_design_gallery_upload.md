# 디자인 갤러리 — 파일 업로드 기능 적용 가이드

관리자 페이지의 **디자인 템플릿 보관함**에 "파일 직접 업로드(드래그앤드롭)" 기능을 추가했습니다.
한 번만 다음 두 가지를 처리해 주시면 됩니다.

## 1. SQL 적용

Supabase 대시보드 → **SQL Editor** → **New query** → 아래 파일 내용 전체 복사 → 붙여넣기 → **Run**

```
supabase/design_studio_gallery_add_upload.sql
```

이 스크립트가 하는 일:

1. `design_gallery_assets` 테이블에 `thumb_url` 컬럼 추가 (썸네일 URL).
2. Storage 버킷 `design_gallery` 생성 (public, 10MB 제한, jpg/png/webp).
3. 인증된 사용자가 객체를 GET 할 수 있는 SELECT 정책.
4. 쓰기는 별도 정책을 만들지 않으므로 Service Role(관리자 API)만 가능.

이미 SQL 을 적용했다면 두 번째 실행해도 안전합니다 (idempotent).

## 2. 사용 흐름

1. 관리자 페이지 → 디자인 템플릿 보관함 관리 → 테마 선택.
2. **"파일로 디자인 추가"** 박스에 이미지를 끌어 놓거나 클릭해서 선택.
3. 브라우저가 자동으로:
   - **인쇄용 원본**: 긴 변 2400px, JPEG 92% (가로/세로 자동 인식, A5/A6 카드 300DPI 충족)
   - **목록 썸네일**: 긴 변 600px, WebP 80% (페이지 로딩용)
4. 두 파일이 `design_gallery` 버킷에 업로드되고 public URL 이 DB 에 저장됩니다.
5. 갤러리 모달은 **목록에 썸네일**, **적용/인쇄 시점에 원본**을 사용합니다.

## 3. 용량 가이드 (참고)

- 1장 평균: 원본 500–800KB + 썸네일 30–60KB → **~600KB / 장**
- 80장 (8 테마 × 10장): **~50MB** (Supabase Free 1GB 의 5%)
- 200장: **~120MB** — 여전히 Free 한도 안

## 4. 외부 URL 추가 (병행 유지)

업로드 박스 아래의 **"또는 외부 URL로 추가"** 영역을 펼치면 기존처럼 `https://...` URL 만으로도 추가 가능합니다.
이 경우 썸네일은 자동 생성되지 않고 동일 URL 을 썸네일로 폴백 표시합니다.
