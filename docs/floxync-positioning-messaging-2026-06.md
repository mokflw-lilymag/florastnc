# FloXync 포지셔닝·메시징 정리 (2026-06)

> 본 문서는 제품·구독·랜딩·베타 운영에 관한 대화를 한곳에 모은 **기획 참고용** 문서입니다.  
> 마케팅 문구는 **코드 기준 팩트**와 구분해 표기했습니다.

---

## 1. 핵심 포지션 (합의 방향)

### 한 줄

**저장만 하는 관리용 프로그램이 아니다.**  
**웹·윈도우·모바일에서 꽃집 사장님을 보필하는 비서 FloXync.**

### 짧은 슬로건 후보

- FloXync — 꽃집 사장님 비서 (웹 · PC · 폰)
- 꽃집에 필요한 인쇄와 정리, FloXync가 대신 해둡니다.
- Not just records. Your shop assistant.

### 톤 가이드

| 하면 좋음 | 피하기 |
|-----------|--------|
| ~해 드릴게요, ~해 둘게요, 「이게 되네」 | RP/PP 브릿지, 8002/8004, XPrint Engine |
| 주문하면 **자동으로 인쇄** | 「말만 하면 AI가 주문 접수」(미구현) |
| 프린터 연결 **도우미** (설정·설치 안내에만) | 일반 랜딩에서 기술 용어 나열 |
| 비서가 **보필**한다 | 만능 AI·네이버/카페24 완전 연동 (과장) |

**일반 사장님 대상 랜딩:** 브릿지 이름을 강조하지 않고 **결과**(자동 인쇄, 리본 출력, 정리)만 보여준다.

## 2. 결제·수익 모델 (팩트)

### 2.1 SaaS 구독 (사장님 → FloXync)

| 구분 | 구현 | 운영 조건 |
|------|------|-----------|
| 한국 | 토스페이먼츠 | 가맹 + `TOSS_*` 환경변수 |
| 해외 | Stripe Checkout (USD) | 가맹 + `STRIPE_SECRET_KEY` |
| UI | `/dashboard/subscription` | 키 없으면 503/안내 |

**전화만으로 끝나지 않음:** PG 가맹 → API 키 → 서버 env → 테스트 → Live 전환. **결제 연동 코드는 이미 있음.**

### 2.2 매장 주문 결제 (고객 → 꽃집)

- 주문 접수 화면에서 **카드/현금/이체/미수 등 수기 기록**.
- **PG 아님.** POS·단말 자동 승인은 `order-payment-gate` 코드만 있고 브릿지 `/payment` **미구현**.

### 2.3 꽃집 ↔ 꽃집 (외주·예치금)

- `external_orders`, `wallets` 코드 **있음**.
- `partner_orders_enabled` 플랫폼 플래그 OFF면 메뉴 비노출.
- **당분간 우선순위 낮음** — 구독자 늘면 검토.

## 3. 사업자 등록 전 베타 전략

| 단계 | 내용 |
|------|------|
| 클로즈드 베타 | 초대 매장, `free` 또는 관리자 수동 `pro` + `subscription_end` |
| 오픈 베타 | 「베타 기간 무료」 명시, 인원·기간 제한 |
| 후원 (선택) | 앱 밖 계좌·링크; 플랜 자동 변경 없음 |
| 정식 전환 | 사업자 + 토스 가맹 → 얼리버드 공지 후 유료 |

**코드 활용:** `free` 플랜, ERP 체험(저장 불가), `/dashboard/tenants` 수동 플랜 부여.  
**미구현:** 구독 페이지 「베타 무료」배너, `BILLING_ENABLED` 결제 버튼 잠금.

## 4. 플랜 구조 (코드 기준 팩트)

### 4.1 플랜 ID

| plan | 표시명 | 결제 |
|------|--------|------|
| `free` | 무료/체험 | ✗ |
| `ribbon_only` | PRINT CORE | ✓ |
| `erp_only` | ERP SMART | ✓ |
| `pro` | FLORA PRO | ✓ |

만료된 유료 플랜 → `free`와 동일 취급.

### 4.2 권한 요약 (`plan-access.ts`)

| | free | ribbon_only | erp_only | pro |
|--|:----:|:-----------:|:--------:|:---:|
| 프린터·디자인 스튜디오 메뉴 | ○ | ○ | ✗ | ○ |
| 리본 출력 | 하루 3회 | 무제한 | ✗ | 무제한 |
| ERP 메뉴 표시 | ○ (체험) | ✗ | ○ | ○ |
| ERP DB 저장 | ✗ | ✗ | ○ | ○ |
| 상품 관리·POS 설정 | ✗ | ✗ | ○ | ○ |

### 4.3 가격 (`pricing.ts`, KRW/월 기준)

- PRINT CORE: 20,000원 (연 200,000원)
- ERP SMART: 30,000원 (연 300,000원)
- FLORA PRO: 50,000원 (연 500,000원)

### 4.4 일반인용 플랜 설명 (랜딩·구독용)

| 플랜 | 한 줄 |
|------|--------|
| PRINT | 리본·영수증 **인쇄** |
| ERP | 주문·배송·지출·마감 **운영** |
| PRO | **인쇄 + 가게 운영** 한 번에 |

## 5. 인쇄 스택 (팩트 — 대외 메시지는 §1 톤으로 변환)

### 5.1 사장님에게 보이는 경험

- 주문(픽업·배송) → **주문서·픽업증·인수증** 등이 카운터 프린터로 **자동 출력**
- 리본 메시지 → 디자인 후 **리본/배너/감열** 프린터로 출력
- 현장(워크인) 주문 → 자동 인쇄 **생략** (의도)
- 다국어 영수증 라벨 지원

### 5.2 내부 구성 (설정·개발 문서용 — 랜딩 비강조)

| 구분 | 포트 | 역할 |
|------|------|------|
| **RP (Ribbon)** | 8002 | 리본·배너·감열 글씨 (Epson 롤 / Xprinter ≤105mm 등) |
| **PP (POS)** | 8004 | 영수증·증표 HTML→PDF→POS (`ppbridge.exe`, v2.6+) |
| **Electron** | — | Windows 앱, 내장 인쇄·오프라인·백업 |

헤더: Electron이면 「데스크톱 인쇄」, 브라우저면 PP ON / RP ON (개발·운영용).

### 5.3 자동 인쇄 흐름

`print-service.ts` → `print_jobs` 큐 (또는 Electron 즉시 `printJob`) → PP 데몬 또는 Electron `printEngine`.

## 6. 지금 되는 기능 vs 랜딩 과장 구간

### 6.1 실제 있음 (ERP: `erp_only`/`pro`, 저장은 유료 ERP만)

- 일정 캘린더, 모바일 웹(픽업·퀵POS·주문 접수)
- 주문·배송·픽업·CRM·지출(OCR 보조)·일일 마감
- 엑셀 가져오기/보내기 (고객·주문·상품·지출)
- Electron 데스크톱, 배송 준비 리마인더
- HQ·다지점 (조직 사용자)

### 6.2 랜딩에 있으나 약하거나 로드맵

- 네이버·카페24 **실시간 연동** (통합 섹션 과장 주의)
- AI 오더 컨시어지 (마케팅 톤 > 실사용 범위 확인 필요)
- 모바일 **스토어 네이티브 앱** — `mobile-premium` 「준비중」이 맞음; **모바일 웹은 이미 있음**
- 매출 엔진: `sns_autopilot`, `flash_sale`, `naver_seo` — 플랜 표에만 있고 UI/API 일부 삭제됨
- 외부몰 스크래핑, VIP 전용관, 실시간 GPS — 구독 페이지 문구와 불일치

## 7. 비서 컨셉 — 일반인 카피 예시

### Hero

**제목:** 꽃집 사장님, 혼자 다 하지 않아도 됩니다.

**부제:** FloXync는 장부만 받는 프로그램이 아니에요. 웹·윈도우·모바일에서 주문·인쇄·정리를 이어 주는 비서예요.

### 비서가 하는 일 (말풍선)

1. 주문 넣으면 **주문서·픽업증·인수증** 알아서 인쇄해 둘게요.
2. **리본 글씨**도 만들고 프린터로 뽑아 드릴게요.
3. **폰**으로도 바로 접수할 수 있어요.
4. **PC** 켜 두면 카운터에서 자동으로 나와요.
5. **지출 영수증** 찍어 두면 장부에 쌓아 둘게요.
6. **오늘 마감**도 한곳에서 볼 수 있어요.

### 3디바이스

| | 역할 |
|--|------|
| 📱 폰 | 현장 접수·픽업·퀵POS (모바일 웹) |
| 🖥 PC | 카운터 자동 인쇄·오프라인 (Windows 앱) |
| 🌐 웹 | 어디서든 주문·배송·지출·마감 |
