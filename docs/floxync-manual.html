왜 브릿지가 필요한가요?
인터넷 브라우저는 보안상 사장님의 컴퓨터에 연결된 하드웨어(프린터)를 직접 제어할 수 없습니다. 그래서 중간에서 명령을 전달해 주는 '브릿지' 프로그램을 매장 PC에 설치해 두어야만 리본과 카드를 척척 뽑아낼 수 있답니다!

1-6. 연동 및 자동화 (쇼핑몰 연결)

네이버 스마트스토어 연동

① 네이버 개발자 센터에 로그인 → 개발자 센터 접속
② ‘Application’ > ‘새 애플리케이션 생성’ → ‘스마트스토어 API’ 사용 신청
③ 발급받은 Client ID와 Client Secret을 복사

④ 플록싱크 환경설정 > ‘연동 및 자동화’ > ‘네이버 연동’ 입력란에 붙여넣고 저장합니다.

※ 토큰은 30일마다 자동 갱신됩니다. 갱신 알림은 ‘알림’ 탭에서 확인하세요.🛈

Cafe24 연동

요약 카페24 개발자센터에서 앱을 만들고 mall.read_order(주문 읽기) 권한을 켠 뒤, Client ID·Client Secret을 복사합니다.

플록싱크 환경 설정 → 연동 및 자동화 → 카드 쇼핑몰 주문 자동 수집 (API 연동)에서 쇼핑몰 아이디(Mall ID)와 키를 넣고 인증 테스트 및 저장 → 카페24 로그인 연동(또는 재인증)으로 OAuth를 마칩니다.

주문 반영 주문 관리 화면 상단 쇼핑몰 동기화(웹)로 카페24 주문을 가져옵니다.

※ 액세스 토큰·리프레시 토큰 주기는 카페24 정책을 따르며, 오랫동안 앱에 접속하지 않으면 재인증이 필요할 수 있습니다. 단계별·FAQ·보안은 저장소 docs/cafe24-integration-guide.md를 보세요.🛈

구글 드라이브 연동

① 구글 계정에 로그인 후 Google Cloud Console 로 이동
② 새 프로젝트 생성 → ‘OAuth 동의 화면’ 설정 → ‘앱 이름’, ‘지원 이메일’ 입력
③ ‘자격 증명’ > ‘OAuth 2.0 클라이언트 ID’ 생성 → ‘리디렉션 URI’에 https://app.floxync.com/auth/google/callback 입력

④ 클라이언트 ID와 클라이언트 비밀키를 복사 후 플록싱크 > ‘연동 및 자동화’ > ‘구글 드라이브 연동’에 입력합니다.

※ 최초 연동 시 구글 권한 요청 팝업이 나타납니다. ‘파일 보기·관리’ 권한을 허용해 주세요.🛈

카카오 알림톡 연동

① 카카오 디벨로퍼스에 로그인 → ‘내 애플리케이션’ → ‘새 애플리케이션 만들기’
② ‘설정’ > ‘플랫폼’ > ‘웹’에 플록싱크 도메인 추가
③ ‘알림톡’ 메뉴에서 ‘채널 추가’ 후 비즈니스 채널을 생성하고, ‘채널 ID’와 ‘채널 시크릿’ 복사

④ 플록싱크 환경설정 > ‘연동 및 자동화’ > ‘카카오 알림톡 연동’에 채널 ID/시크릿 입력 후 저장.

※ 알림톡 템플릿은 ‘템플릿 관리’에서 미리 등록해 두어야 합니다.🛈

구글 시트 자동 연동

① 구글 드라이브 연동과 동일한 OAuth 인증을 사용합니다.
② 플록싱크 > ‘연동 및 자동화’ > ‘구글 시트 연동’에서 연동할 시트 URL을 입력합니다.
③ ‘자동 동기화 설정’ 켜기 → 주문이 발생하면 실시간으로 시트에 행이 추가됩니다.

※ 시트는 ‘Floxync Orders’라는 이름으로 자동 생성됩니다. 필요 시 직접 시트를 편집해도 됩니다.🛈

2 주문 관리: 똑똑한 AI 비서와 함께하기

사장님, 이제 수첩에 주문을 적으실 필요 없어요.

2-2. 수동 주문 입력 (전화/방문 주문)

🚛

📝
리본 문구

리본에 뽑을 글자를 오타 없이 적어주세요.

5-0. 메뉴 지도 (한눈에)

새 주문

경로: /dashboard/orders/new (PRO·ERP 플랜)

개요 제목은 보통 새 주문 등록이며, 기존 주문을 열면 주문 수정, 협력사 발주 흐름이면 협력사 발주 (상호명)으로 바뀝니다.

화면 구성 왼쪽(넓은 화면 기준)에는 고객·상품·수령/배송 섹션이 세로로 이어지고, 오른쪽에는 주문 요약·금액·저장이 모입니다. 상단 오른쪽에 AI 주문 도우미(컴포넌트)가 붙어 있어 문자·카톡 내용을 붙여 넣고 필드를 채울 수 있습니다.

순서 ① 고객 검색 또는 신규 입력 → ② 상품·수량·맞춤 상품 → ③ 수령 방식(매장수령·수령예약·배송예약)과 일시·주소 → ④ 리본/카드 문구 등 메모 → ⑤ 요약에서 결제 수단·할인 확인 → ⑥ 저장. 모바일에서는 요약 패널이 접히므로 하단 버튼으로 다시 열 수 있습니다.

막힘 해소 배송비가 이상하면 환경 설정의 배송비 규칙과 본 화면의 주소 입력이 일치하는지 확인하세요.

주문 목록

경로: /dashboard/orders

개요 페이지 제목은 주문 관리입니다. 설명은 터치 UI일 때와 PC일 때 문구가 조금 다릅니다(검색·필터·카드 탭 등).

상단 버튼 일일 마감 정산(/dashboard/orders/daily-settlement), 엑셀, 구글 시트, 웹에서 쇼핑몰 동기화(Cafe24 등), 새 주문 등록이 한 줄에 있습니다.

필터 통계 카드 아래에서 상태(전체·준비중·완료·취소), 수령 유형(배송·수령예약·매장수령), 기간(2개월~전체), 집계 기준(수령일 vs 주문일)을 고릅니다.

행 메뉴: 인쇄 및 출력

각 주문 행의 점 세 개(또는 동일 역할) 메뉴에서 다음을 실행합니다. 상세 모달을 연 뒤 하단의 카드 출력·리본 출력은 같은 동작으로 이어집니다.

  • 주문서 인쇄 — 주문 미리보기 인쇄 화면(/dashboard/orders/print-preview/주문ID)으로 연결됩니다. 진행 중 토스트로 준비 상태를 알려 줍니다.
  • 카드 메시지 출력 — 먼저 출력 도구 선택 모달이 뜹니다. 기존형 카드는 /dashboard/design-studio?orderId=…&target=card, 폼텍 라벨은 target=formtec로 디자인 스튜디오에 들어가 카드·라벨 편집·출력 흐름을 이어 갑니다.
  • 리본 출력 — 주문의 메시지·보내는 분·주문자명을 바탕으로 좌·우 문구를 나눈 뒤 /dashboard/printer?left=…&right=… 리본 프린터 화면으로 이동합니다. 안드로이드 앱으로 접속한 경우에는 이 메뉴가 숨겨질 수 있으니 웹에서 이용해 주세요.

폼텍 라벨지 칸·폰트·시작 칸 등을 세밀히 고르는 메시지 인쇄 미리보기 전용 화면(/dashboard/orders/print-message)도 앱에 포함되어 있습니다. 목록의 카드 메시지 출력은 기본적으로 디자인 스튜디오 쪽으로 연결됩니다.

주문 수정

행 메뉴의 주문 수정을 누르면 주문 수정 다이얼로그가 열립니다. 주문자·연락처·수령 방식·일정·수령인·주소·상품 줄·메시지(카드/리본)·결제 상태·메모·실제 배송비·배송비 결제 수단 등을 한 화면에서 고치고 저장할 수 있습니다. 복잡한 전체 재작성은 새 주문 화면에서 URL에 기존 주문 ID를 붙여 여는 방식과 병행해 쓰는 매장도 있습니다.

막힘 해소 안드로이드 앱 모드에서는 쇼핑몰 동기화·새 주문 등록 버튼이 숨겨질 수 있습니다. 웹 브라우저에서 접속해 보세요.

일일 마감 정산

경로: /dashboard/orders/daily-settlement · 주문 목록 상단에서도 진입

목적 선택한 정산일(reportDate)을 기준으로 그날 들어온 매출·수금·미결제·금고(현금) 흐름을 한 번에 보고, 금고 잔액을 확정해 일일 정산 마감 버튼으로 저장합니다. 상단 주문 목록으로 돌아가기로 언제든 목록으로 나올 수 있습니다.

날짜와 데이터 범위

좌우 화살표와 날짜 입력(type="date")으로 정산일을 바꿉니다. 날짜를 바꾸면 그날까지 포함해 최근 주문·당일 지출 등을 다시 불러옵니다. 분할 결제나 예전 주문의 당일 입금은 이월 입금 쪽 집계에 반영됩니다.

상단 요약 카드 네 장

  • 금일 총 매출액 — 당일 실제 수금으로 잡힌 합계에 더해, 환경 설정의 매출 집계 기준이 주문일이면 같은 날짜의 주문 건 총액으로 카드에 보이는 방식이 달라질 수 있습니다(결제일 기준과 구분).
  • 수금 (현금 + 계좌) — 현금·계좌이체로 집계된 수금 합계이며, 카드 결제액은 부가 설명으로 같이 표시됩니다.
  • 금고 예상 잔액 — 아래 금고 흐름 카드의 수식으로 계산된 최종 마감 예정 잔액과 연동됩니다.
  • 금일 주문 접수 — 해당일 주문일 기준 접수 건수와, 그중 미결제 건수·미수 금액 요약입니다.

금고 현금 상세 흐름

이월 잔액은 전일 저장된 정산 기록이 있으면 자동 이어 붙이고, 없으면 직접 숫자를 넣을 수 있습니다. 수정저장으로 이월 금액만 먼저 고정해 둘 수 있습니다.

  • 이월된 현금 잔액 — 전일 마감 시점 금고에서 이어진 값(또는 수동 입력).
  • 금일 현금 매출 — 당일 집계된 현금 매출(직접 수금).
  • 배송비 현금 지급액 — 주문에 기록된 현금 배송비와, 지출 중 배송·운송으로 잡힌 현금을 비교해 더 큰 쪽을 반영해 빠짐을 줄입니다.
  • 기타 현금 지출 — 당일 지출 중 현금(배송비 중복 제외) 합계.
  • 관리자 시재 입금/출금 — 은행 맡기기 등 금고에서 빠지거나 들어온 조정액을 직접 입력합니다.
  • 최종 마감 예정 잔액 = 이월 + 금일 현금 매출 − 시재 입금/출금 − 배송비 현금 − 기타 현금 지출.

일일 정산 마감을 누르면 위 금고 관련 숫자와 날짜가 일일 정산 저장 API로 넘어가며, 성공 시 토스트로 저장되었음을 알립니다. 저장 필드 개념은 previous_vault_balance, cash_sales_today, delivery_cost_cash_today, cash_expense_today, vault_deposit 등으로 이해하시면 됩니다.

표와 목록

  • 금일 결제 처리된 주문 내역 — 해당일에 수금이 잡힌 주문(당일 접수 건 + 예전 주문의 당일 입금)을 시간·번호·주문자·수단·총액·금일 입금액으로 표시합니다. 분할 결제의 두 번째 결제가 오늘이면 금일 입금액 칸에 두 번째 금액이 나옵니다.
  • 금일 미결제(미수금) 목록 — 그날 주문일인 주문 중 아직 결제가 끝나지 않은 건을 모아 미수 점검에 씁니다.
  • 오늘 발생한 지출 요약 — 운송비(현금)·기타 지출(현금)·카드 포함 총 지출과 건수, /dashboard/expenses로 가는 링크가 있습니다.
막힘 해소 총매출 카드 숫자가 통계 화면과 다르면 환경 설정의 매출 집계 기준(주문일 vs 결제완료일)을 먼저 맞춰 보세요. 금고 잔액은 현금·계좌 수금과 현금 지출 중심이므로 카드 매출만으로는 금고가 맞지 않을 수 있습니다.

고객 CRM

경로: /dashboard/customers

단골을 이름으로만 기억하지 말고, 구매 이력·포인트·서류까지 한 화면에서 이어 가세요.

개요 화면 제목은 고객 관리입니다. 검색창으로 이름·연락처·회사명을 찾고, 표에서 행을 클릭하면 넓은 고객 상세 다이얼로그가 열립니다. 여기서 주문을 다시 열어보거나, 법인 거래처에 보낼 거래명세서·개인에게 드릴 간이영수증·상담용 견적서까지 바로 발행할 수 있습니다.

목록 · 상단 도구

  • 새로고침 — 최신 고객·주문 연동을 다시 불러옵니다.
  • 디지털 서류함 — 온라인으로 발행한 견적서·거래명세서·영수증 발행 이력을 모아 둡니다(안내 문구상 최대 30일 보관 후 폐기되므로, 필요하면 브라우저 인쇄 대화상자에서 PDF로 저장하세요).
  • 엑셀보내기, 신규 고객 등록 — 명단 백업과 신규 카드 등록.

상세 창 한눈에

상단에는 등급·POS 연동 배지와 함께 총 구매액·마지막 구매일·보유 포인트 요약 카드가 붙습니다.

아래 탭은 주문 이력(최신순, 행 클릭 시 주문 상세 모달) · 포인트 내역(적립·사용·취소, 주문/POS 출처 배지) · 상세 정보(연락처·메모 등)로 나뉩니다.

고객 클릭 후 — 거래명세서 · 간이영수증 · 견적서

상세 창 오른쪽 상단에 세 가지 서류 버튼이 나란히 있습니다. 간이영수증과 거래명세서는 같은 2단계 마법사를 쓰고, 문서 종류만 다릅니다. 견적서는 별도 견적 다이얼로그로 이어집니다.

간이영수증 — 수령·증빙용 영수 느낌 거래명세서 — 기간·품목 나열형 명세

1단계 · 발행 항목 선택

  1. 귀하(수신) 블록에서 회사명·담당자명을 고객에게 맞게 다듬습니다. 연락처·이메일은 이후 인쇄 페이지 헤더에도 반영됩니다.
  2. 공급자(발행) 블록은 환경 설정의 사업자 정보(상호, 등록번호, 대표, 주소, 연락처)와 테넌트명을 불러와 채웁니다. 비어 있으면 기본 placeholder가 보이므로, 실제 발행 전에 설정을 먼저 맞추는 것이 좋습니다.
  3. 조회 시작일·종료일로 주문을 가져올 기간을 정합니다. 처음 열면 보통 이번 달 1일 ~ 오늘에 가깝게 잡혀 있습니다.
  4. 아래 목록은 해당 기간 안에서 주문자 연락처가 이 고객 카드의 연락처와 동일한 주문만 조회합니다. 목록이 비면 기간을 넓히거나, 주문서에 적힌 번호에 하이픈 유무가 다른지 확인해 보세요.

2단계 · 최종 확인 및 인쇄

  • 하단 안내: 발행 기록은 관리 목적으로 30일간 보관된다는 문구가 있습니다. 장기 보관이 필요하면 PDF 저장(브라우저 인쇄 → PDF)을 권장합니다.

3. 출력 프린터와 브릿지

  1. 출력 프린터 라벨 아래 녹색/빨간 점이 브릿지 연결 상태를 보여 줍니다. 오른쪽에 Bridge Engine v… 버전이 표시됩니다.
  2. 드롭다운: 브릿지가 프린터 목록을 주면 기기명이 뜹니다. 목록이 비면 「매장 기본 프린터로 원격 전송」 같은 예비 옵션이 보일 수 있습니다.
  3. 같은 줄 오른쪽 새로고침(↻) 아이콘을 누르면 127.0.0.1:8002/api/printers 를 다시 읽습니다. 반응이 없으면 브릿지 미실행 또는 방화벽입니다.
  4. 패널 하단의 「브릿지 연결 문제 해결 및 초기화」 를 누르면 안내 모달이 열립니다(작업표시줄 R 아이콘, 8002 포트, 설정 초기화 등).

4. 리본 프리셋 · 폭 · 길이 · 여백

  • 리본 프리셋을 바꾸면 폭·길이·레이스·상하단 여백이 한꺼번에 맞춰집니다. Xprinter 계열을 고르면 라벨에 ≤105mm 안내가 붙고, 선택 가능한 프리셋 목록이 좁아질 수 있습니다.
  • 폭(Width mm) · 길이(Length mm)는 숫자로 미세 조정합니다.
  • 상단여백: 헤드 시작부터 글자까지(mm). 하단여백: 글자 끝에서 다음 용지까지(mm). 양쪽레이스: 리본 양끝 여백(mm).
  • 수평(좌우) 보정 슬라이더로 ±2mm 단위로 미세 이동합니다. 실제 기기마다 틀어짐을 여기서 상쇄합니다.

5. 인쇄 대상 · 용지 모드 · 품질

  • 인쇄 대상: 양쪽 모두 / 경조사어(왼쪽만) / 보내는이(오른쪽만).
  • 컷 리본이 기본입니다. 롤 리본을 고르면 현재는 개발 중 알림이 뜨고 다시 컷으로 돌아갑니다.
  • 고속 인쇄 vs 고급(저속) 중 선택합니다. 컷 리본일 때는 커팅 길이 셀렉트 대신 커팅 NO 박스가 보입니다.

6. 좌·우 리본 글꼴과 편집

  • 각 열 제목: 좌측 리본 (경조사), 우측 리본 (보내는이). 입력창을 누르면 해당 열이 활성됩니다.
  • 폰트 마법사: 한글/한자/영문/기호 탭으로 나뉘어 있고, 드롭다운에서 글꼴을 고릅니다. B 버튼으로 굵게, 가로% · 세로%로 글자 비율을 맞춥니다.
  • 캔버스에서 글자를 클릭하면 90도 회전 토글이 적용되는 글자 단위 편집이 됩니다(구현상 회전 상태는 좌/우 각각 유지).
  • 자주 쓰는 문구: 카테고리 셀렉트 + 그리드 버튼. 누르면 현재 활성 쪽 텍스트가 통째로 바뀝니다. 옆 톱니는 문구 관리 다이얼로그입니다.
  • 특수 기호 그리드에서 기호를 누르면 커서 위치에 삽입됩니다.

7. 로고 · 템플릿 · 미리보기 · 인쇄

  1. 내 점포 로고: 점선 박스를 눌러 이미지를 올리고, 오른쪽 토글로 인쇄 시 로고 합성 여부를 켭니다.
  2. 하단 바의 Templates: 저장된 리본 구성을 불러옵니다.
  3. Preview를 누르면 인쇄 미리보기 레이아웃으로 바뀌고, 버튼 라벨이 Design으로 바뀌어 다시 편집 모드로 돌아갑니다.
  4. 배율 숫자(%)는 캔버스 확대·축소이며, 실제 인쇄 해상도와는 별개입니다.
  5. PRINT: 구독/권한 체크를 거친 뒤 브릿지로 데이터를 보냅니다. 브릿지가 없으면 설치 안내 모달이 뜹니다.
  6. 사이드바 맨 아래 Ribbonist 초보자 매뉴얼 버튼을 누르면 앱 내장 도움말 다이얼로그가 열립니다.

  • 주문 흐름에서 링크로 들어오면 ?orderId=UUID 가 붙습니다. 이때 한 번만 주문 데이터를 읽어 리본/메시지 문구를 텍스트 블록으로 올리고, 받는이·보내는이 토글을 켭니다. 같은 탭에서 새로고침해도 중복 배치되지 않도록 세션 키로 잠금합니다.
  • ?target=formtec 이면 3114(8칸) 폼텍 프리셋으로 맞추고 칸 선택 모드로 시작합니다. 그렇지 않으면 기본 카드 규격 A5 210×148mm 로 시작하며 내지 면을 먼저 엽니다.

2. 상단 헤더 (항상 고정)

  • 제목: PROFESSIONAL STUDIO (터치 UI에선 부가 문구가 줄어듭니다).
  • 탭: 앞면/뒷면 (Cover) 과 내지/안쪽 (Inside) — 접이 카드일 때 앞뒤와 안쪽 면을 오갑니다.
  • 디자인 저장 버튼: 클라우드에 저장하며 성공 시 토스트 「디자인이 클라우드에 저장되었습니다.」
  • 프린터 아이콘: 폼텍 프리셋이면 먼저 폼텍 셀 선택 모달이 열립니다. 일반 용지면 고해상도 PDF를 만들고 브라우저 인쇄 창을 띄웁니다. 진행 중 「고해상도 다중 페이지 PDF를 생성하고 있습니다…」 로딩 토스트가 뜹니다.
  • 우측 상단(데스크톱): 현재 용지 가로×세로 mm 표시가 나옵니다.

3. 왼쪽 사이드바 (도구 류)

위에서 아래로 주요 블록입니다.

  1. 텍스트 레이어 추가 — 캔버스에 새 텍스트 박스(기본 문구 「새 텍스트」)를 올립니다. 접이(반페이지) 표지에서는 기본 위치가 뒷표지 쪽 중앙에 가깝게 잡힙니다.
  2. PNG·이미지 레이어 추가 — 파일 선택기로 이미지를 올립니다.
  3. 매직 스튜디오 (배경 제거 & 편집) — 별도 사진 편집 모달을 엽니다.
  4. 디자인 템플릿 보관함 — 저장해 둔 배경/템플릿을 불러옵니다. 설명대로 배경만 가져온 뒤 텍스트는 위 버튼으로 다시 얹는 방식이 안전합니다.
  5. 용지 규격 및 방향 (접기): 프리셋 셀렉트는 카드 규격과 폼텍 라벨 그룹으로 나뉩니다. A5, A4, 엽서, 폼텍 3130·3102·3639·3114·3112 등. 그 아래 가로형/세로형, 단면/접이, 접이 시 접지 가이드라인 표시 토글.
  6. 추천 메시지 찾기 / 명언 라이브러리 — 감동·축하 등 테마별 문구 모달.
  7. 내지 + 접이 모드일 때만: 수발신자 자동 배치 — To. 수령인 / From. 발신인 토글과 입력칸.
  8. 선택 요소 속성 편집 — 캔버스에서 텍스트나 이미지를 클릭한 뒤, 여기서 글 내용·글자 크기·색·회전·줄간격·자간·폰트 패밀리·외곽선·정렬·레이어 순서 등을 바꿉니다. 이미지면 파일 교체와 마법의 주문 사진 편집기 버튼이 보입니다.
  9. 상점 브랜딩 자동 배치 — 로고·상호·전화·웹사이트를 한 번에 깔끔히 올립니다(환경설정·테넌트 정보 기반).

Marketing · 자동화

AI 홍보 마스터

개발 중

경로: /dashboard/marketing

꽃 손질·배송·주문에 쓰다 보면 인스타·숏폼·블로그는 늘 내일로 밀립니다. 이 메뉴는 사장님 매장을 위한 24시간 자율 마케팅 비서를 표방하며, 한 번 로그인해 두고 SNS 연결·브랜드 DNA·자동 모드만 맞춰 두면, 캡션·긴 글·틱톡·쇼츠용 대본 등 콘텐츠 초안과 (연동된 채널로의) 게시까지 AI가 대신 짚어 줍니다. 영상 자체의 제작·편집은 채널·API 범위 안에서 단계적으로 넓혀 가는 구조로 이해하시면 됩니다.

역할

꽃만 만지는 사장님의
마케팅 매니저

약속

전략 → 카피 → 검수 에이전트가 협업하는 스튜디오형 UI와, DNA 탭의 자율 주행(Auto-Pilot) 스위치로 “매일 자동 생성·업로드” 방향을 화면에 명시합니다.

사전 준비

각 채널 연결(자동 연결 또는 수동 토큰 입력)이 되어 있어야 “대신 올리기”가 가능합니다. 인스타그램은 안내에 따라 비즈니스 계정 전환이 자동 업로드에 유리하다고 적혀 있습니다.

화면 구성 — 세 개의 탭

① 통합 관제

상단 카피대로 24시간 자율 마케팅 비서가 “작동 중”인 느낌을 주는 대시보드입니다. 왼쪽에는 전략 기획·콘텐트 작가·품질 검수 세 에이전트 카드가 있고, 클릭하면 오른쪽에 미션·파이프라인·브랜드 가이드라인이 펼쳐집니다.

  • AI 홍보 작전 시작하기 — 버튼 한 번으로 오케스트레이션이 돌아가며, 진행 바와 단계 배지(기획·원고·검수)가 보입니다. 완료 후 플랫폼별 초안(제목·본문)이 카드로 뜨고, 준비가 되면 이대로 포스팅하기로 게시 API를 호출합니다.
  • 과거 기록 영역(카드)으로 발행 히스토리를 모아 볼 수 있는 자리가 있습니다.

② SNS 연동 관리

Instagram, Facebook, Threads, YouTube Shorts, TikTok, Google Blogger, Naver Blog 등 채널별 연결 상태를 한눈에 보고, 자동 연결 또는 수동 입력으로 자격 증명을 등록합니다. TikTok·YouTube 등은 OAuth로 브라우저 이동 후 돌아오는 흐름이 붙어 있습니다.

오른쪽 어두운 카드에는 “관리자 앱 등록 → 사용자 일회 로그인 → AI가 토큰을 들고 파이프라인으로 전송” 같은 완전 자동화 원리 설명이 있습니다. 사장님 입장에서는 연결만 해 두면 된다는 메시지로 이해하시면 됩니다.

③ 브랜드 DNA 설정

페르소나(우아함·따뜻함·트렌디·전문가 등)을 고르고, 홍보 주제 문장을 여러 개 등록해 두면 AI가 글감을 고를 때 참고합니다. 대상 SNS는 스위치로 instagram / youtube / naver / tiktok / threads 중 어디에 올릴지 고릅니다.

자율 주행 마케팅(Auto-Pilot) — “24시간 자동 모드” 스위치를 켜면, 안내 문구대로 매일 트렌디한 주제를 골라 설정된 페르소나·주제에 맞춰 콘텐츠를 자동 생성하고 각 SNS에 업로드한다고 표시됩니다. 끄면 저장만으로는 자동 실행되지 않는다고도 적혀 있습니다. 맨 아래 DNA 설정 및 자동화 저장으로 shop_settings에 반영합니다.

사장님이 하실 일 — 세 줄 요약

  1. 로그인해 두고, 마케팅 메뉴에 들어옵니다.
  2. SNS 연동과 브랜드 DNA(페르소나·주제·대상 채널)를 저장합니다.
  3. 바쁠 땐 Auto-Pilot ON으로 맡기고, 여유 있을 땐 AI 홍보 작전 시작하기로 초안을 보고 수정·포스팅합니다.
현실적인 안내 자동 업로드는 채널 API·토큰·매장 정책에 따라 달라질 수 있습니다. 글이 안 올라가면 SNS 탭의 연결 상태와 화면 토스트 메시지를 먼저 확인하고, 필요 시 수동 포스팅으로 초안만이라도 활용해 보세요. 본 메뉴는 완제품이 아니므로 업데이트 노트를 수시로 확인해 주세요.
