# 최신 버전(ERP V4) 바코드 및 자재관리 시스템 기획안

본 문서는 원본 ERP 시스템(`lilymagerp-v4_supa`)에서 검증된 최신 실무 기반 바코드 및 자재관리 시스템의 기획을 플로싱크(Floxync) 환경으로 가져와 정리한 내용입니다. 과거의 복잡한 AI 분석 및 터치식 차감 대신, 현장에서 가장 빠르고 정확하게 사용할 수 있는 스캐너 기반 실무형 모델로 대체되었습니다.

## 1. 대/중분류 키워드 조합형 실무 바코드 (Compact Barcode)
과거의 긴 난수형 UUID 기반 바코드를 폐기하고, 물리적인 폼텍 라벨지(특히 소형)에 여백 없이 인쇄될 수 있도록 바코드 규격을 축소 및 실무화했습니다.

* **바코드 규격 제한:** 최대 10자 이내 (스캐너 인식 및 라벨지 크기 제약 준수)
* **네이밍 컨벤션:** 카테고리의 대분류와 중분류 앞글자(키워드)를 추출한 3글자 + 짧은 식별 번호 (예: `FLO-1234`)
* **기대 효과:** 직원이 라벨지에 인쇄된 바코드 문자열만 육안으로 확인해도 대충 어떤 계열의 자재인지 즉시 파악 가능.

## 2. 하드웨어 스캐너 연동 직접 입출고 (Direct Scanner Input)
AI 카메라 분석 차감 기능 대신, 정확도 100%를 자랑하는 바코드 스캐너 연동 기능을 핵심으로 둡니다.

* **UI 위치:** `/dashboard/barcode-scanner` (바코드 스캐너 전용 페이지 신설)
* **작동 방식:** 
  1. 매장 직원이 자재에 부착된 바코드를 스캐너로 리딩 (또는 키보드로 바코드 번호 직접 입력)
  2. 시스템에서 즉시 해당 자재/상품의 정보 패널을 팝업
  3. 현재 재고 확인 및 입/출고 수량을 그 자리에서 바로 증감 반영

## 3. 직관적인 자재 입/출고 및 본사 요청 프로세스
수동 터치식 차감의 번거로움을 없애고, 주기적인 재고 실사와 본사 직접 요청에 집중합니다.

* **재고 직접 수정 (`/dashboard/materials`):** 현재 보유 중인 자재를 리스트나 검색으로 찾은 뒤 정확한 입출고 기록(사유 포함)을 남기도록 유도합니다. 오차를 방지하기 위해 사용자가 확정 기록을 남기는 것을 원칙으로 합니다.
* **본사 자재 요청 (`/dashboard/material-request`):**
  1. 지점 매장에 특정 자재가 부족할 경우 바코드 리딩 또는 검색으로 해당 자재를 선택.
  2. 필요한 수량과 요청 사유, 우선순위(긴급 등)를 지정하여 클릭.
  3. 본사 시스템으로 즉시 발주 알림이 전송되며, 처리 상태(대기-승인-배송-완료)를 지점에서 트래킹 가능.

## 핵심 요약
> "작은 라벨지에 쏙 들어가는 대/중분류 혼합형 10자리 콤팩트 바코드"를 인쇄하여 자재에 부착하고, "바코드 스캐너를 쏴서 즉각 재고를 조회하고 본사에 자재를 요청(발주)하는 직관적이고 가장 정확한 실무 방식"으로 기획의 방향이 최종 수정 및 확정되었습니다.
